## A · Diseño / UI

### 🔴 P0
- [ ] **Logo: ajustar tamaño y peso en header negro.** Hoy en `Logo.tsx` el
  tamaño es 56px (cuadrado). Como el PNG es 1092×1095 con padding interno
  transparente, el wordmark visible queda chico. Probar 64–80 y validar en
  desktop/mobile · `src/components/layout/Logo.tsx`

### 🟡 P1
- [ ] **Tipografía de marca**. Hoy se usa la fuente del sistema (Geist/sans).
  Para joyería de autor recomiendo combinar una serif elegante para títulos
  (Cormorant, Söhne Halbfett, EB Garamond) con la mono actual para etiquetas ·
  `src/styles/tokens.css` + `src/app/layout.tsx`
- [ ] **Cards del catálogo: mostrar estado de stock**. El status (sold out,
  pre-order) aparece en carruseles pero no en los tiles. Para vitrine sin
  carrito, mostrar el estado en la card es clave UX comercial ·
  `src/components/catalogue/ProductCard.tsx`
- [ ] **Footer denso de marca**. Hoy solo IG/FB/Contact + copyright. Sumar
  dirección (Vilniaus g. 30), email, teléfono, horarios y línea legal · 
  `src/components/layout/Footer.tsx`
- [ ] **Hairline del header sobre hero negro**. Con header noir + hero noir el
  borde inferior puede desaparecer; subir opacidad a `rgba(255,255,255,.12)` o
  agregar shadow sutil para definir el corte.

### 🟢 P2
- [ ] Hover-swap en cards: solo dispara si hay 2ª foto — validar visualmente.
- [ ] Animación de aparición (`Reveal`) puede acumularse en mobile; revisar
  delays en cascada.
- [ ] Detalle: la `<dl>` de specs tiene `borderTop` con `rgba(0,0,0,.1)` —
  inconsistente si la página entra en modo dark a futuro; tokenizar.

## B · UX

### 🔴 P0
- [ ] **Focus management del Lightbox**. Al abrir el zoom fullscreen el foco
  debe ir al botón de cerrar; al cerrar, volver al thumbnail/main. Hoy no se
  gestiona, atrapa a usuarios con teclado · `ProductGallery.tsx`
- [ ] **El detalle de producto sin descripción tira mensaje vacío**. Los 21
  buckets genéricos (ring 1, floral, earring 2, etc.) muestran `<p>` vacío
  donde va la descripción. Renderizar fallback o ocultar el bloque ·
  `src/app/catalogue/[id]/page.tsx`

### 🟡 P1
- [ ] **Sticky aside vs header**. El detalle usa `top: 100` para el aside.
  Tras el cambio el header mide ~94px — verificar que no se superpongan en
  scroll · `src/app/catalogue/[id]/page.tsx:83`
- [ ] **CategoryNav en mobile**. Si supera el ancho, debe scrollear horizontal
  con `scroll-snap` o colapsar a menú · `src/components/layout/CategoryNav.tsx`
- [ ] **Link "Atelier" del header → /about**. La etiqueta sugiere taller, pero
  /about mezcla historia + contacto. Separar en 2 ítems del header (*Atelier*
  → historia, *Contacto* → mailto/mapa) o renombrar el link a *Studio*.
- [ ] **a11y del LocaleSwitcher**. Confirmar `aria-pressed`/`aria-current` en
  el idioma activo y foco visible · `src/components/layout/LocaleSwitcher.tsx`
- [ ] **Cookies sin consentimiento (GDPR)**. `rb-locale` se setea sin banner.
  EU exige al menos un aviso para cookies no estrictamente necesarias (las
  funcionales como locale suelen estar exentas, pero conviene un microaviso).
- [ ] **404 page**. Validar que `not-found.tsx` quedó con `<Header />` sin
  prop (ya hecho) y que las imágenes/copy hagan sentido.

### 🟢 P2
- [ ] Skip-link de teclado al `<main>` (ya hay uno mencionado en git log
  "skip link" — verificar que sigue presente).
- [ ] Sin búsqueda — con 52 piezas no es crítico, agregar cuando >100.
- [ ] Wishlist/favorites: no aplica para vitrine; descartar.

## C · Copy

### 🔴 P0
- [ ] **21 piezas con nombre placeholder visibles** (`ring 1`, `floral`,
  `earring 2`, `unassigned1`, etc.). Para una marca de autor esto destruye la
  percepción de cuidado. Reconectar a piezas reales con descripción (Claude
  ya tiene la clasificación visual) o quitarlas del CSV hasta nombrarlas ·
  `content/products.csv`
- [ ] **Sección "Coming Soon" contradice la realidad**. Dice "We're opening a
  studio-store…" pero la dirección Vilniaus g. 30 ya está pública. Opciones:
  - eliminar `ComingSoon` del home,
  - **o** reescribir como *"Visit the studio"* (ya hay datos: dirección, horas,
    by-appointment) · `messages/*.json → comingSoon` + `src/components/sections/ComingSoon.tsx`

### 🟡 P1
- [ ] **Hero eyebrow mezcla idiomas en EN**. `"SS 26 · Žiedlapiai"` queda raro
  en la versión en inglés. Sugerencias: `"SS 26 · Petals"` o mantener el LT
  pero acompañado · `messages/en.json → home.heroEyebrow`
- [ ] **Status taxonomy luxury-friendly**. *"Pre-order"* es lenguaje retail.
  Para joyería de autor: *"Made to order"*, *"Reserve"*, *"One-of-a-kind"*.
  Revisar con Rima · `messages/*.json → status.*`
- [ ] **Detalle: subject del mailto** (`detail.enquireSubject`) — revisar que
  use bien `{name}` y `{code}` en ambos idiomas.
- [ ] **Lithuanian native review**. Las traducciones LT deberían pasar por un
  hablante nativo (legales, técnicas como peso/aleaciones, hospitalidad).
- [ ] **Materiales repetitivos** ("Sidabras 925, Auksas" en muchas piezas).
  Pedir a Rima detalle por pieza (14k? 18k? rosé?) — sube percepción de valor
  · `content/products.csv`

### 🟢 P2
- [ ] *"ENQUIRE"* en mayúsculas con tracking ancho funciona, pero podría ser
  *"Enquire about this piece →"* para claridad.
- [ ] Footer copyright fijo en 2026 — automatizar (`new Date().getFullYear()`).
- [ ] Sumar microcopy en thumbnails de galería (`fig. 01`, `02`…) que ya está,
  validar legibilidad sobre fondos oscuros.
